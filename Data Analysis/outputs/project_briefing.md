# Napps Monthly Brand Performance Report — Project Briefing

## What is this?

An automated system that generates monthly performance reports for ~50 ecommerce brands on the Napps platform. Every month, it pulls data from the Napps API, scores each brand, generates AI-powered insights, and delivers a PDF report per brand + an Excel summary of all brands.

## The Problem

Napps has ~50 brands on their platform. Every month, someone needs to:
- Pull performance data for each brand
- Understand what's going well and what's not
- Compare brands against each other and against industry benchmarks
- Write up recommendations for each brand
- Package it all into professional reports

Doing this manually for 50 brands is impractical. This system automates the entire process.

## What the Reports Include

### Per-Brand PDF Report (one per brand, ~5 pages)
1. **Cover page** — brand name, month, overall performance flag (Good / Average / Poor)
2. **Executive Summary** — AI-written overview of the brand's performance + metrics-at-a-glance table
3. **Metric Deep-Dives** — each metric gets a card with its value, peer rank, and an AI-written narrative explaining what the number means and why it matters
4. **Improvement Recommendations** — 3-5 prioritised, actionable suggestions generated by AI based on the data
5. **Appendix** — raw data table with all values and peer rankings

### All-Brands Excel Workbook (one file, 3 sheets)
1. **Overview** — one row per brand, all metrics, colour-coded flags (green/yellow/red)
2. **Recommendations** — all AI recommendations across all brands, filterable by priority
3. **Raw Data** — plain numbers for further analysis

## The 11 Metrics Tracked

| Metric | What it measures |
|---|---|
| Total Sales | Revenue for the month |
| Conversion Rate | % of sessions that result in an order |
| AOV (Average Order Value) | Average spend per order |
| Add-to-Cart Rate | % of sessions where a product is added to cart |
| Cart Abandonment Rate | % of carts that don't convert to orders |
| CLV (Customer Lifetime Value) | Average total revenue per customer |
| Churn Rate | % of customers who don't return |
| Sessions / Traffic | Number of visits to the store |
| Refund / Return Rate | % of orders refunded |
| Repeat Purchase Rate | % of customers who buy more than once |
| Total Orders | Number of completed orders |

## How Scoring Works

Each metric is scored using up to 4 signals:
- **Sector threshold** (35%) — is the value good/poor for this brand's sector (fashion, beauty, electronics, etc.)?
- **Industry benchmark** (35%) — how does it compare to universal ecommerce benchmarks?
- **MoM trend** (15%) — is it improving or declining vs. last month?
- **Peer rank** (15%) — where does this brand sit among all 50 brands?

Each metric gets a flag: **Good**, **Average**, or **Poor**.

The overall brand flag is a weighted average of all metric flags, with conversion rate and sales weighted most heavily.

## How the AI Insights Work

After scoring, each brand's data is sent to Claude (Anthropic's AI) via API. Claude receives:
- All metric values, flags, and peer ranks
- Month-over-month trends
- Sector and benchmark context

Claude then generates:
- An executive summary (3-4 sentences)
- A narrative for each metric (2-3 sentences explaining what the number means)
- 3-5 prioritised improvement recommendations with explanations

This means every brand gets personalised, data-driven analysis — not generic templates.

## The Pipeline (How It Runs)

```
Step 1: Fetch data from Napps API (one call per brand)
Step 2: Merge & process metrics, compute MoM deltas
Step 3: Score each metric and assign flags
Step 4: Call Claude API to generate insights (~50 API calls)
Step 5: Render PDF per brand (WeasyPrint + HTML/CSS template)
Step 6: Render Excel workbook for all brands
Step 7: Save snapshot for next month's MoM comparison
Step 8: Organise reports into output folder
```

The full pipeline runs with a single command and is scheduled to run automatically on the 2nd of each month.

## Tech Stack

- **Python** — all scripts
- **Napps REST API** — data source for all metrics
- **Claude API (Anthropic)** — AI-generated insights and recommendations
- **WeasyPrint + Jinja2** — PDF rendering from HTML/CSS templates
- **openpyxl** — Excel workbook generation
- **Google Sheets** — brand configuration and historical snapshots
- **Google Drive** — report delivery

## Architecture: WAT Framework

The system follows a **Workflows, Agents, Tools** architecture:

- **Workflows** (markdown files) — define what to do, step by step
- **Agents** (Claude) — make decisions, handle errors, orchestrate
- **Tools** (Python scripts) — do the actual work deterministically

This separation keeps AI focused on reasoning while code handles execution, making the system reliable and repeatable.
