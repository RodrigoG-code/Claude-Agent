Phase 1 — Data Collection
1
Fetch Napps Data
fetch_napps.py
Connects to the Napps REST API for each active brand and downloads all ecommerce data for the target month.
a
Authenticate — Uses OAuth2 (client ID + secret) to get an access token from the Napps auth server
b
Request metrics — Calls the analytics endpoint with the date range and brand ID
c
Save raw data — Writes one JSON file per brand to the temporary folder
7 metrics collected per brand:
Total Sales Conversion Rate AOV Add-to-Cart Rate Cart Abandonment CLV Churn
Output: .tmp/raw/napps/{brand}_YYYY_MM.json — one file per brand (~50 files)
raw JSON files
Phase 2 — Data Processing
2
Process & Compute Deltas
merge_metrics.py
Takes the raw data from Step 1, loads last month's saved snapshot, and calculates how each metric changed compared to the previous month.
a
Load raw files — Reads all the brand JSON files from .tmp/raw/napps/
b
Load prior month — Reads last month's snapshot from data/snapshots/ (if it exists)
c
Calculate MoM % — For each metric: ((this month - last month) / last month) × 100
Example
If last month's conversion rate was 3.0% and this month is 3.6%, the MoM change = +20%
First run?
No prior data exists yet, so all MoM deltas will be empty. This is normal — they appear from month 2 onwards.
Output: .tmp/processed/metrics_YYYY_MM.json — unified file with all brands + deltas
metrics + deltas

Phase 3 — AI Analysis
4
Generate AI Insights
generate_insights.py
Sends each brand's scored data to Claude AI (Anthropic's language model). The AI acts as a senior ecommerce analyst and writes a personalised analysis for each brand.
a
Build prompt — Formats the brand's metrics, flags, trends, and peer ranking into a structured prompt
b
Call Claude API — Sends the prompt to claude-sonnet-4-6 (~50 API calls, one per brand)
c
Parse response — Extracts the structured JSON with summaries and recommendations
AI generates 3 things per brand:
Executive Summary
2-3 sentence narrative overview highlighting the overall flag, best metric, and biggest concern
Metric Narratives
2-3 sentences per metric explaining what the number means in context
Improvement Recommendations
3-5 prioritised actions (High/Medium/Low) with specific, practical advice for the brand's sector
Output: Insights added to peer_ranks_YYYY_MM.json (same file, enriched)
enriched data
Phase 4 — Report Generation
5
Render PDF Reports
render_pdf.py
Generates a professional 5-page A4 PDF report for each brand, using an HTML/CSS template converted to PDF by WeasyPrint.
5 pages per PDF:
1
Cover page — Brand name, period, overall flag badge, executive summary quote
2
Executive summary — Full narrative + metrics-at-a-glance table with flags and peer ranks
3
Metric deep-dives — Each metric with value, trend, thresholds, benchmarks, and AI narrative
4
Recommendations — AI-generated improvement actions, sorted by priority
5
Appendix — Raw data table, data sources, and any quality warnings
Output: .tmp/reports/pdf/{brand}_YYYY_MM.pdf — ~50 PDF files
6
Render Excel Workbook
render_excel.py
Creates a single Excel file with all brands side-by-side, designed for managers who want to compare, filter, and sort across the entire portfolio.
3 sheets in the workbook:
1
Overview — All brands with colour-coded flag cells (green/yellow/red), auto-filters, frozen header
2
Recommendations — AI recommendations per brand, sorted by priority
3
Raw Data — All numeric values, MoM changes, scores, and peer ranks
Output: .tmp/reports/excel/brand_performance_YYYY_MM.xlsx
report files
Phase 5 — Storage
7
Save Historical Snapshot
save_snapshot.py
Saves this month's processed data as a local snapshot file. This is what enables Month-over-Month comparisons — next month's Step 2 will read this file to calculate deltas.
How it works
Month 1: no prior data, no deltas. Month 2: reads Month 1's snapshot, calculates changes. Month 3: reads Month 2, etc.
Data grows over time
Each month adds one snapshot file. After 12 months you have a full year of history for trending.
Output: data/snapshots/YYYY_MM.json
8
Organise Reports
organize_reports.py
Moves all generated PDFs and the Excel workbook from the temporary folder into a clean, date-stamped output directory that's easy to find and share.
Output: outputs/YYYY_MM/ — final folder with all deliverables
